RocketMQ下载安装和配置

RocketMQ的Binary版是一些编译好的jar和辅助的shell脚本，可以直接从官网下载

安装条件：

  • 64位操作系统，Linux / Unix / Mac;
  • 64位JDK 1.8+;
  • 4G+ 空闲磁盘用于Broker服务器
  • Maven 3.2.x;
  • Git;

注意：如果需要从GitHub上下载源码和编译的话，需要安装Maven3.2.X和Git

源代码下载地址（须构建为二进制后启用）：
https://www.apache.org/dyn/closer.cgi?path=rocketmq/4.4.0/rocketmq-all-4.4.0-source-release.zip

二进制版本下载地址（免构建）：
http://rocketmq.apache.org/release_notes/release-notes-4.4.0/

构建操作（二进制版本跳过本操作）

> unzip rocketmq-all-4.4.0-source-release.zip
  > cd rocketmq-all-4.4.0/
> mvn -Prelease-all -DskipTests clean install -U
> cd distribution/target/apache-rocketmq

启动NameServer

> nohup sh bin/mqnamesrv &
  > tail -f ~/logs/rocketmqlogs/namesrv.log
The Name Server boot success...

启动brocker

> nohup sh bin/mqbroker -n localhost:9876 &
> tail -f ~/logs/rocketmqlogs/broker.log 
The broker[%s, 172.30.30.233:10911] boot success...

关机服务器

> sh bin/mqshutdown broker
The mqbroker(36695) is running...
Send shutdown request to mqbroker(36695) OK

> sh bin/mqshutdown namesrv
The mqnamesrv(36664) is running...
Send shutdown request to mqnamesrv(36664) OK

附录

find (目录) [-type d | f] (文件夹 | 文件) -name （名称，可使用正则表达式）

find / -name “runserver.sh”

RocketMQ启动出现内存不足的解决方法

按照官网启动rocketMQ会造成ｊａｖａ运行内存的不足．需要修改找到runserver.sh和runbroker.sh，

找到runserver.sh修改里面的
JAVAOPT=”JAVAOPT−server−Xms128m−Xmx256m−Xmn256m−XX:PermSize=128m−XX:MaxPermSize=320m”

runbroker.sh这个文件原来是这样的JAVAOPT=”{JAVA_OPT} -server -Xms8g -Xmx8g -Xmn4g”

修改成
JAVA_OPT=”${JAVA_OPT} -server -Xms256m -Xmx256m -Xmn128m”
